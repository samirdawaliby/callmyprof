1. Platform Overview

CallMyProf is an international tutoring platform connecting students with qualified tutors. The platform supports multiple languages (English, French, Arabic with RTL), handles the complete student journey from lead capture to video sessions, and provides comprehensive admin tools for managing the business.

Business Flow

  1. Student sees an ad (TikTok, Facebook, Instagram, Google)
  2. Fills out the form on callmyprof.com
  3. A lead is created in the CRM
  4. Admin contacts and qualifies the need
  5. Booking email sent → student picks a time slot
  6. Payment → auto-generated video link → tutoring session

5. All Updates & Changes (March 29, 2026)

All changes listed below were implemented, deployed, and verified on production on March 29, 2026.

5.1 Foreign Key Fix (cours_eleves)

ItemDetail
ProblemCreating a session at /cours/new failed with "FOREIGN KEY constraint failed"
Root Causecours_eleves.eleve_id had a FK referencing the empty eleves table, but student IDs came from the users table (role='eleve')
FixMigration 0011_fix_cours_eleves_fk.sql recreated the table without the invalid FK constraint
Code ChangesAll queries changed from JOIN eleves to JOIN users WHERE role='eleve'
Files Modifiedsrc/index.js (getCoursFormData, cours detail query, tutor students query)

Auto-Backfill Logic

When the admin sessions page (/sessions) or group classes page (/group-classes) loads, it checks for cours sessions with missing video rooms. If found and the session date is in the future, it automatically creates a Daily.co room and updates the database.

8. Student & Tutor Portals

Student Portal

  • Dashboard with upcoming sessions count, next session, total paid
  • Sessions page merging bookings + admin-created cours
  • Video Join buttons for upcoming sessions
  • Payment history & methods
  • Profile editing with avatar upload
  • Security (password change)
  • Support ticket system

Tutor Portal

  • Dashboard with class stats and earnings
  • Classes page with calendar view + Action column
  • Video Join buttons (host URL with host privileges)
  • Schedule/availability management
  • Student list from assigned sessions
  • Earnings & payout settings
  • Support ticket system

Data Architecture Note

Two Student Systems: Students exist in the users table (role='eleve') for portal access AND historically in the eleves table (legacy CRM, now empty). All current code references users table. The cours_eleves.eleve_id column references users.id (FK constraint was removed in migration 0011).
